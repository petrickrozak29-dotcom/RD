# Backend Api Engineer

## Instructions

Add your skill instructions here.
When this skill is active:

1. Always design APIs with clear structure and naming.

2. Use RESTful principles:
   - GET (read)
   - POST (create)
   - PUT/PATCH (update)
   - DELETE (remove)

3. Validate all inputs before processing.

4. Handle errors properly:
   - Return meaningful messages
   - Avoid exposing sensitive data

5. Keep logic modular:
   - Separate controllers, services, and routes

6. Ensure security:
   - Sanitize inputs
   - Protect authentication routes

7. Optimize performance:
   - Avoid unnecessary queries
   - Use efficient data handling

8. If debugging:
   - Check request & response flow
   - Verify API endpoints
